# 会议纪要：神州数码武汉团队商机对齐

**时间**：2026年5月12日
**参会人**：我、高松
**客户及关键人**：神州数码武汉交付团队，负责人：侯浩

---

## 一、 客户现状与背景信息
- **合作意愿**：客户合作态度积极，参与度高。一方面是交付团队确有提升效率的刚需，另一方面也面临来自上层要求提效的压力。
- **核心痛点**：陷入“伪效能”困境。个人工作量饱和且绩效可见，但整个组织的整体产出和交付效率并没有明显提升。
- **触动因素**：观察到兄弟部门（ITS团队）在使用效率工具后，团队吸活速度和整体效能实现了 30%-40% 的可见提升，因此产生了强烈的对标和改进需求。

## 二、 客户核心诉求
侯浩目前明确提出了以下三个诉求：
1. **效能提升方案**：希望获取能够平移的效率提升方案，帮助其团队实现组织级的效能跨越。
2. **探究 AI Native 组织**：好奇并希望了解作为 AI Native 公司的我们，内部是如何组织运作和开展日常工作的。
3. **跟班体验学习**：提出能否派几名员工来我司“结对/跟班”工作几天，实地体验并学习先进的干活经验和方式。

## 三、 预算与商务情况评估
- **预算规模**：目前评估该商机金额在 **20万 - 30万** 左右。
- **采购流程**：据了解，30万以下的金额可能无需走复杂的招采流程，侯浩作为团队负责人应具备资金支配权。
- **高层支持**：该类提效项目契合神码上层的战略要求。只要一线交付团队（侯浩方）意向明确、觉得好用，上层推进阻力较小。

## 四、 应对策略与解决方案思路
针对客户的诉求，我们内部对齐了初步的应对和推进策略：

1. **暂缓“来我司跟班学习”的诉求**：
   - 考虑到内部工作方式的保密性、难以直接平移复制，以及缺乏标准化的对外接待流程，直接让客户过来体验并不现实，需先往后推。
2. **主打策略：将 20-30 万预算转化为“试点团队 (Pilot Team)”项目**：
   - **咨询切入**：前期以咨询战略切入，不只是卖工具。
   - **深入赋能**：提供 FDE (前端部署工程师) 的深入赋能，教他们如何真正用好工具。
   - **驻场陪跑**：可安排 FDE 出差驻场约一周时间，与客户团队在一块实际工作，手把手教。
   - **价值**：通过打造一个成功的试点团队，不仅能跑通模式，打磨我们自己的交付材料，还能为客户后续在内部跨团队复制提供范本。
3. **坚持“不打无准备之仗”**：
   - 坚决避免在情况不明时直接派人去现场摸底，成本太高。
   - 在 FDE 进场或去现场实施前，必须先通过远程访谈等形式，摸透客户一线员工的具体工作方式、核心痛点，并带着一套成型的方案再过去落地。

## 五、 下一步行动计划 (Action Items)

- [ ] **高松**：负责与侯浩对接，预约本周四或周五进行一次线上沟通。

  - 进行 High-level 沟通，进一步挖掘客户的真实痛点和业务现状。
  - 向侯浩演示目前我们其他客户的成功案例和工作方式（Case Study）。
  - 借演示机会试探客户反馈，引导其接受“打造试点团队”的合作模式。
- [ ] **内部资产沉淀（我方）**：
  - 梳理并设计标准化的客户调研问卷（需包含 Stakeholder 是谁、项目发起人、核心目标等）。
  - 借此机会将从 0 到 1 的商机接入流程规范化，为后续打磨成型的实施材料做准备。
