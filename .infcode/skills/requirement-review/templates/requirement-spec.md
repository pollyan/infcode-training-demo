## 功能需求（Functional Requirements）

| ID | 需求描述 | 强度 | 验收标准（Acceptance Criteria） |
|----|---------|------|-------------------------------|
| FR-001 | | SHALL | |
| FR-002 | | SHOULD | |

> **强度说明：**
> - `SHALL` / `MUST`：绝对要求，不可妥协
> - `SHOULD`：强烈推荐，有充分理由可不做但必须说明
> - `MAY`：可选，由 Developer 决定是否实现
> - `MUST NOT`：明确禁止的行为
>
> **验收标准写法要求（重要）：**
> - 必须是可独立验证的具体行为
> - ✅ 正确示例："当用户提交空白表单时，系统返回错误提示'请填写必填项'"
> - ❌ 禁止写法："功能正常运行"、"系统应该支持 XX"
