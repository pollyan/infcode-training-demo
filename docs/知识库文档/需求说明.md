# 需求说明：同步外部档案

> 版本：2.0 | 日期：2026-05-20 | 状态：已批准
> 详细实施规格见：[requirement-spec-客户档案同步](../specs/requirement-spec-客户档案同步.md)

---

## 业务背景与目标

销售运营、交付支持和客户成功团队在使用客户主数据列表页时，常发现客户状态、联系电话等关键信息滞后于上游客户中心的更新，导致跟进基于过时数据做出判断。

本需求在现有列表页每行操作列新增"同步外部档案"按钮，让销售人员可主动触发一次从客户中心到本地客户主数据的单向校验与更新，恢复数据新鲜度信任。

**目标用户**：销售团队（销售人员、销售运营人员）

**成功标准**：数据正确同步并展示；单条同步 5 秒内完成；失败后可重试

---

## 术语约定

采用项目领域词汇表（CONTEXT.md）统一术语：

| 术语 | 含义 | 禁止替代词 |
|------|------|-----------|
| 客户主数据 | CRM 中客户核心档案信息集合 | ✗ 客户信息、客户资料 |
| 同步外部档案 | 从客户中心单向拉取最新资料更新本地客户主数据 | ✗ 拉取数据、刷新数据、同步数据 |
| 客户中心 | 上游外部系统，客户档案的权威数据源 | ✗ 外部系统、第三方 |
| 同步状态 | 同步操作结果标识（SUCCESS / PENDING / FAILED） | ✗ 同步结果、同步标记 |

## 功能需求

### FR-1：新增同步按钮

客户主数据列表页每行操作列新增"同步外部档案"按钮，与现有操作按钮并列显示。页面保留筛选区、统计卡和客户列表不变。

### FR-2：按钮状态管理

- 点击后按钮立即进入禁用状态，文本变为"同步中..."
- 同步完成（成功或失败）后按钮恢复可点击，文本恢复为"同步外部档案"
- 超时后文本显示"重试"，点击后重新发起完整同步流程
- 不同行的同步按钮互不影响，可同时对不同行发起同步

### FR-3：后端同步流程

后端收到同步请求后：
1. 调用客户中心接口（携带 `X-App-Key: xingchen-crm-local` 请求头）查询对应客户的最新资料
2. 将客户中心返回的外部字段映射为内部 DTO，禁止将外部原始响应直接透传到前端
3. 更新客户主数据的客户名称、客户状态、联系电话、更新时间、同步状态五个字段
4. 同步成功时同步状态更新为 SUCCESS；失败时更新为 FAILED

### FR-4：行级局部更新

同步完成后仅更新当前行数据（行级局部刷新），禁止触发整页重载或重新加载列表，保障用户翻页/筛选的工作上下文不丢失。

### FR-5：成功反馈

同步成功时页面顶部展示"客户档案同步成功"提示消息，当前行的五个字段值更新为最新值。即使外部数据与当前数据完全相同，仍视为同步成功并刷新更新时间。

### FR-6：失败反馈与错误映射

后端捕获客户中心原始错误后，转换为用户可理解的中文提示再返回前端，禁止透传原始错误码、英文信息或堆栈跟踪：

| 外部场景 | 前端展示 |
|---------|---------|
| 客户编号不存在（4004） | "同步失败：客户档案不存在" |
| 参数校验失败（4001） | "同步失败：请求参数异常" |
| 上游熔断 / 不可用（5000） | "客户中心服务暂时不可用，请稍后重试" |
| 并发限流（5002） | "客户中心服务暂时不可用，请稍后重试" |
| 连接失败 / HTTP 5xx | "客户中心服务暂时不可用，请稍后重试" |
| 前端 5 秒超时 | "同步超时，请点击重试"（按钮变为"重试"） |

### FR-7：数据脱敏

联系电话必须在后端完成脱敏后再返回前端，脱敏规则：手机号保留前 3 后 4，中间 4 位用星号屏蔽（如 `138****5678`）。禁止将脱敏责任下放到前端。

## 非功能需求

| 需求 | 指标 |
|------|------|
| 同步响应时间 | 后端调用客户中心并返回结果，P95 < 3 秒，P99 < 5 秒 |
| 前端超时阈值 | 5 秒未收到响应视为超时 |
| 页面局部更新 | 仅更新当前行，不刷新整页 |
| 后端外部调用超时 | 调用客户中心超时阈值 10 秒；失败重试最多 3 次，指数退避（1s、2s、4s） |
| 数据脱敏 | 联系电话后端脱敏后再返回前端 |

---

## 明确排除

- **批量同步**：仅支持单条客户记录同步
- **权限控制**：所有用户均可操作，无需角色或权限验证
- **定时自动同步**：仅手动触发，不涉及定时任务
- **同步历史记录**：不记录同步操作日志或审计信息
- **WebSocket / SSE 实时推送**：客户端等待服务端响应，不采用异步回调
- **新增页面或路由**：在现有列表页内完成所有交互
