# 前端架构与 UI 组件交互规范

为了规避中大型系统的“技术腐化”并保障 UI 交互的一致性体验，制定以下前端技术架构红线规范。

## 1. 数据请求与集成约束
- **禁止私自构造原生客户端**：在所有的视图容器组件内部（如列表入口组件中）绝对禁止直接编写 `fetch()` 或 `axios.get()` 裸请求发起调用网络。
- **隔离收口**：涉及所有与后端联通的交互逻辑，必须抽象并定义在统一的 `src/services/` (或类似作用域) 目录下。React/Vue 的视图组件内只能引用 Service 层的实例方法。

## 2. 交互反馈规范 (Message & Notification)
- **禁用原生弹窗**：所有的 Web/App JS代码中禁止出现原生的 `alert()`、`confirm()` 以及 `prompt()` 语法。
- **统筹提示分发**：操作和业务状态提示，必须依赖封装好的统一通知组件。
  - 成功时展示：如 `Message.success("成功描述")` 或项目中集成的等价库（如 Antd / Element 的相关唤起方式）。
  - 失败时展示：如 `Message.error("失败原因描述")` 发挥强提示警示效果。

## 3. 页面无损渲染策略
- **不可销毁式重载**：在任何单页操作完成后，禁止使用 `window.location.reload()` 这种暴力刷新整页浏览器的操作方式毁灭局部状态树。
- **局部状态更新机制**：对于表格等行级操作，应当充分运用 React/Vue 内置的单向数据流或内部状态库逻辑。直接用接口回传的最新的数据来浅拷贝或合并复写关联的那条行内状态信息数据（Partial State Mutation），实现毫无停滞感觉的非阻塞界面变更。
