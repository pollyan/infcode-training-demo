# 业务全景与产品需求文档 (PRD)：客户档案同步

## 1. 业务背景与痛点
当前，销售运营、交付支持和客户成功团队在使用本 CRM 系统的“客户主数据列表页”时，经常遇到以下痛点：
客户信息（如客户状态、联系人电话等）在“星辰集团客户中心（Master Data）”修改后，本地系统存在数据滞后。这导致销售和支持人员无法第一时间联系到客户或处理业务。
为了解决这个数据割裂的问题，我们需要在现有的列表页中，提供一种轻量、及时的“手动同步”能力。

## 2. 领域名词定义 (Ubiquitous Language)
- **主数据 (Master Data)**：指集团视角的权威客户档案信息。
- **客户中心 (Customer Center)**：负责维护主数据的上游业务系统。
- **本地档案 (Local Profile)**：当前 CRM 模块内部保存的客户台账副本。
- **同步 (Sync)**：拉取上游客户中心最新数据，并覆盖更新本地档案中对应字段的过程。

## 3. 功能需求详细说明
### 3.1 界面交互变更
在现有的**客户主数据列表页**的每一行客户记录的**操作列**中，新增一个名为 **“同步外部档案”** 的操作动作。

### 3.2 交互反馈要求
- **正在处理 (Loading)**：点击按钮后，应当采取适当的前端防抖或 Loading 展现，防止连续触发多次请求。
- **成功反馈**：拉取并更新完成后，在页面弹出极简的成功提示（如：“同步成功”）。
- **无刷新局部更新**：**只重载与刷新当前被操作的数据行**（包括客户名称、状态、联系电话、更新时间和同步状态等字段），坚决不允许触发整个页面的全量重载，以保障用户原本由于翻页或筛选产生的工作上下文不丢失。
- **失败反馈**：当同步失败时，需提供友好的错误原因提示（如：“同步失败：客户档案不存在”或“系统繁忙请稍后再试”）。

## 4. 业务容错规则
- 外部接口如果缺失了部分非必填字段（例如没有返回 `contact_phone`），本地落库与展示时不得将其强制覆盖为空白，需做合理的空值保护。
