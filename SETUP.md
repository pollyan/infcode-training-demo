# SETUP

练习项目代码库地址：

- GitHub: `https://github.com/pollyan/infcode-training-demo`

## 1. 环境要求

### 前端与 mock 服务

- Node.js 20+
- npm 10+

### 后端

- Java 17+
- Maven 3.9+

## 1.1 讲师预演原则

- 讲师预演时应尽量对齐客户真实 IDE 和版本
- 讲师预演时应尽量对齐客户真实工程打开方式
- 讲师预演时应尽量模拟客户真实网络和权限条件
- 预演不仅要验证功能流程，也要验证文档上传、知识库构建、代码分析和调试链路

## 2. 启动顺序

建议按以下顺序启动：

1. 启动 mock 外部服务
2. 启动后端服务
3. 启动前端页面

## 3. 推荐直接用脚本

在项目根目录运行：

```bash
./scripts/check-env.sh
./scripts/start-all.sh
```

停止服务：

```bash
./scripts/stop-all.sh
```

> 说明：培训与预演时均建议从工程根目录打开项目，并确认当前项目已初始化 Git，以便插件正确识别 `.infcode` 目录及代码知识能力。

## 4. 启动命令

### 4.1 启动 mock 外部服务

```bash
cd mock-server
npm install
npm start
```

启动后访问：

`http://127.0.0.1:9090/health`

### 4.2 启动后端服务

```bash
cd backend
mvn spring-boot:run
```

启动后访问：

`http://127.0.0.1:8080/api/health`

### 4.3 启动前端页面

```bash
cd frontend
npm install
npm start
```

启动后访问：

`http://127.0.0.1:5173`

## 5. 训练建议顺序

### 如果你要做客户演练

建议优先使用 `main` 分支。

```bash
git checkout main
```

它保留了真实列表页骨架，但没有补完“同步外部档案”能力，适合让学员现场完成。

### 如果你要看参考答案

建议使用 `done` 分支。

```bash
git checkout done
```

它包含一份完整参考实现，适合讲师备课、对照或兜底。

当前仓库中的参考实现包含：

## 8. 培训时推荐演示流程

1. 建立上下文
2. 做需求理解
3. 输出实施计划
4. 先做后端
5. 再做前端
6. 再做测试与排错
7. 最后做 Diff 审核

## 9. 讲师提示

- 如客户使用真实项目培训，应优先在真实项目环境中验证主流程
- 如客户项目为 Java 多模块工程，应提前确认工程打开方式；当前建议从项目根目录打开完整工程，不建议以 workspace 或局部模块方式打开
- 如现场出现环境级阻塞，应将其记录为培训输出，而不是简单忽略
- `VS Code` 仅作为必要时的兜底方案，不作为培训默认假设
